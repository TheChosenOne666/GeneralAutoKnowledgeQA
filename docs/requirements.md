# 熊答 — 需求文档

> 基于 RAG + Agent 的多租户智能问答平台，让企业知识真正流动起来。

## 1. 产品概述

### 1.1 产品名称

**熊答**（BearAnswer）

### 1.2 产品定位

面向企业的多租户智能知识问答平台。新员工秒懂公司规范，告别重复沟通；基于 RAG + Agent 技术，回答精准可溯源。

### 1.3 核心价值

- **混合检索 + 重排序**，回答精准可溯源
- **多租户 RBAC**，数据隔离安全合规
- **ReAct Agent 多步推理**，复杂问题也搞定

---

## 2. 角色与权限

### 2.1 角色定义

| 角色 | 说明 | 数量 |
|---|---|---|
| 普通成员（member） | 租户内普通用户，可问答、管理个人知识库 | 多 |
| 租户管理员（tenant_admin） | 租户内管理者，可管理共享知识库、AI配置、成员、审计日志 | 少 |
| 平台超管（super_admin） | 平台运营方，可管理所有租户、平台配置、全局审计 | 极少 |

### 2.2 权限矩阵

| 功能模块 | 普通成员 | 租户管理员 | 平台超管 |
|---|---|---|---|
| 问答（对话） | ✅ | ✅ | ✅（体验） |
| 共享知识库 | 只读（可问答） | 完整管理（增删改） | — |
| 个人知识库 | 完整管理（仅自己） | 完整管理（仅自己） | — |
| AI 模型配置 | ✅ 可配置 | ✅ 可配置 | — |
| 成员管理 | ❌ | ✅ | — |
| 审计日志 | ❌ | ✅（本租户） | ✅（全局） |
| 租户管理 | ❌ | ❌ | ✅ |
| 平台配置 | ❌ | ❌ | ✅ |

### 2.3 菜单可见性

| 菜单项 | 普通成员 | 租户管理员 | 平台超管 |
|---|---|---|---|
| 对话 | ✅ | ✅ | ✅ |
| 知识库 | ✅ | ✅ | — |
| 搜索 | ✅ | ✅ | — |
| AI 模型配置 | ✅ | ✅ | — |
| 成员管理 | ❌ | ✅ | — |
| 审计日志 | ❌ | ✅ | ✅ |

> **注**：普通成员也能配置 AI 模型（选择使用哪个 LLM/Embedding），但不能管理成员和查看审计日志。

## 3. 功能需求

### 3.1 认证模块

#### 3.1.1 登录

- 邮箱 + 密码登录
- 密码可见切换（眼睛图标）
- 社交登录：GitHub、Google
- 登录后跳转到问答页

#### 3.1.2 注册

- 姓名 + 邮箱 + 密码 + 确认密码
- 密码强度检测（4格强度条，实时变色：红/橙/黄/绿）
- 协议勾选（用户协议 + 隐私政策）
- 注册后自动登录跳转

#### 3.1.3 视觉要求

- Canvas 粒子系统（90个粒子 + 距离连线 + 鼠标交互）
- 极光光晕（Emerald + Teal 渐变流动）
- 毛玻璃卡片
- 浅绿主题

### 3.2 问答模块

#### 3.2.1 首页布局（WeKnora 风格）

- **左侧边栏**（240px）：
  - Logo + 网站名"熊答"
  - 图标菜单：对话、知识库、搜索、AI模型配置、成员管理、审计日志
  - 历史会话列表（近7天分组）
  - 底部用户信息（头像 + 姓名 + 角色）
- **中间主区域**：
  - 顶部大标题："基于知识库内容问答 – AI 问答"
  - 副标题："你可以这样问我"
  - 推荐问题胶囊（横向排列，点击直接提问）
  - 底部悬浮输入框
  - 知识库选择标签
  - 工具栏：智能推理下拉、图片按钮、附件按钮、绿色+号、模型选择

#### 3.2.2 对话功能

- SSE 流式输出，实时显示回答
- 引用溯源：每条回答标注来源文档（文件名 + 页码 + "查看原文"按钮）
- ReAct Agent 多步推理：复杂问题分解为多个步骤
- 模式切换：知识库问答 / 网络搜索
- 上下文记忆：支持多轮对话
- 导出对话记录

#### 3.2.3 会话管理

- 新建会话
- 历史会话列表（按时间分组）
- 会话搜索
- 删除会话
- 会话重命名

### 3.3 知识库模块

#### 3.3.1 知识库分类

| 类型 | scope 字段 | 维护者 | 可见性 |
|---|---|---|---|
| 共享知识库 | `shared` | 租户管理员 | 全员可问答，普通成员只读 |
| 个人知识库 | `personal` | 个人用户 | 仅自己可见可问答 |

#### 3.3.2 知识库管理页

- **Tab 切换**：共享知识库 / 我的知识库
- 每个Tab显示文档数量角标
- 共享库提示："共享知识库由租户管理员维护，全员可问答；普通成员只读"

#### 3.3.3 文档管理

- **上传**：拖拽上传，支持 PDF / Word / Markdown / TXT
- **状态追踪**：上传 → 解析 → 向量化 → 就绪 / 失败
- **进度条**：实时显示处理进度
- **文档列表**：表格展示（文件名、类型、大小、状态、上传时间、操作）
- **操作**：查看详情、重新处理、删除
- **筛选**：按类型、状态筛选
- **搜索**：按文件名搜索

#### 3.3.4 文档处理流程

```
上传文件 → 文本提取 → 分块（Chunking）→ 向量化（Embedding）→ 存储（向量库）
                                                                    ↓
                                                              就绪（可用于问答）
```

### 3.4 AI 模型配置模块

#### 3.4.1 配置项

| 模型类型 | 说明 | 示例 |
|---|---|---|
| LLM | 大语言模型，用于生成回答 | 火山方舟·豆包Pro / deepseek-v3.2 |
| Embedding | 向量化模型，用于文档/查询编码 | 火山方舟·Doubao-embedding |
| Rerank | 重排序模型，用于精排检索结果 | 可选 |

#### 3.4.2 配置页布局

- 顶部状态栏：显示当前 LLM / Embedding / Rerank 配置状态（已配置/未配置）
- 配置卡片（2列网格）：
  - LLM 配置卡：提供商选择、模型选择、API Key、温度参数、最大Token
  - Embedding 配置卡：提供商选择、模型选择、API Key、向量维度
  - Rerank 配置卡：提供商选择、模型选择、API Key
- 保存配置按钮

#### 3.4.3 权限说明

- 普通成员可配置自己使用的模型偏好
- 租户管理员可配置租户级默认模型

### 3.5 成员管理模块

#### 3.5.1 成员列表

- 表格展示：头像、姓名、邮箱、角色、状态、加入时间、最后活跃
- 搜索成员
- 筛选：按角色、状态筛选

#### 3.5.2 成员操作

- 邀请成员（邮件邀请）
- 修改角色（member ↔ tenant_admin）
- 停用/启用成员
- 移除成员

#### 3.5.3 权限矩阵展示

- 可视化展示各角色权限对照表

### 3.6 审计日志模块

#### 3.6.1 日志列表

- 表格展示：时间、用户、操作类型、操作详情、IP地址
- 操作类型徽章：登录/登出、文档上传/删除、配置修改、成员变更等
- 展开查看 JSON 格式的详细操作数据

#### 3.6.2 筛选

- 时间范围筛选
- 操作类型筛选
- 用户筛选
- 关键词搜索

#### 3.6.3 权限说明

- 租户管理员：查看本租户所有操作日志
- 平台超管：查看全局所有租户操作日志

## 4. 非功能需求

### 4.1 性能

- 问答首字延迟 < 2s（SSE 流式）
- 文档上传后向量化完成 < 30s（10MB 以内）
- 页面加载 < 1.5s

### 4.2 安全

- 多租户数据隔离（行级 / Schema 级）
- API 鉴权（JWT + RBAC）
- 敏感操作审计记录
- API Key 加密存储

### 4.3 可用性

- 响应式设计（适配桌面/平板）
- 浅绿主题，视觉一致性
- 输入框聚焦光晕、按钮悬停动效等微交互

## 5. 术语表

| 术语 | 说明 |
|---|---|
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| Agent | 基于 LLM 的智能体，可多步推理 |
| RBAC | Role-Based Access Control，基于角色的访问控制 |
| 多租户 | Multi-tenant，一套系统服务多个租户，数据隔离 |
| 共享知识库 | 租户级公共知识库，全员可问答 |
| 个人知识库 | 用户私有知识库，仅自己可见 |
| SSE | Server-Sent Events，服务器推送流式输出 |
| Chunking | 文档分块，将长文档切分为语义段落 |
| Rerank | 重排序，对检索结果精排提升准确率 |
